Exercise 1 [10 Points]: Print the first n characters

Write a C program that takes a string and an integer value as command-line arguments and prints the first n characters of the string, where n is the integer value provided as argument. If n is greater than the length of the string, print an error message.

$ ./print_first_n_characters "Hello, world\!" 5
Output: Hello

$ ./print_first_n_characters "Good morning" 12
Output: Good morning

$ ./print_first_n_characters "Good morning" 20
Invalid value for n.

Hint: you can use strlen() function from <string.h> to find the length of the string. It should use atoi() function from <stdlib.h> to convert the integer value from string to integer.

Important: The functionality that show the n first characters of the string should be defined as an auxiliary functions

Exercise 2 [10 Points]: Average and float types

Write a C program that calculates the average of numbers. The program should accept input from either the command line or standard input. If command line arguments are provided, the program should compute the average of those numbers. If no command line arguments are provided, the program should prompt the user to enter numbers via standard input until the user signals the end of input (e.g., by pressing Ctrl+D or Ctrl+Z).

Examples:

./average 1 2 3 4 5 6
Average: 3.50
./average
Enter numbers (press Ctrl+D to end):
1
2
3
4
5
Average: 3.00

Exercise 3 [15 Points]: Diagonal matrix

A diagonal matrix is defined as a square matrix where all elements outside the main diagonal are zero. The main diagonal consists of elements where the row index equals the column index (i.e., positions matrix[i][i]).

Write a C program that manipulates matrices of maximum size 30 x 30. The program should receive the size of the matrix as an integer from the command line, initialize the matrix with values from standard input, display the original matrix, determins if the given matrix is diagonal, and finally display a message indicating whether the matrix is diagonal or not.

Important: The functionalities that shows a matrix and checks if it is diagonal should be defined as auxiliary functions

./transpose 3
1 0 0 0 2 0 0 0 0
Original Matrix:
1   0   0
0   2   0
1   0   0

The matrix is diagonal

Exercise 4 [10 Points]: Square root

Write a function that takes an integer n as input and computes the largest integer x such that x^2 is less than or equal to the given number n. The search for the number should not be linear; that is, you cannot start from 0 and increment by one, nor start from n or n/2 and decrement by one. The approach used should be binary search.

Write a main program that reads the number from standard input, invokes the function, and shows the result.

Exercise 5 [15 Points]: Ternary search algorithm

Develop a ternary search algorithm inspired by binary search. Instead of splitting the array into two parts, it divides the array into three sections to pinpoint the potential location of the desired value.

A separate function should implement the algorithm. The main function should initialize an array with values obtained from standard input. It should validate that the entered values are sorted; if not, it should display an appropriate error message. Subsequently, it should call the search function and display the result.

Exercise 6 [20 Points]: Couloured binary search

Create a program named printbinarysearch.c that utilizes ANSI Escape Codes to visually demonstrate the binary search algorithm’s progression. The program should accept command-line arguments specifying the size n of the array to be used and the number to be searched. It initializes the array with the first n even numbers, then conducts a binary search. During each iteration of the binary search algorithm, a suitable auxiliary function printBinSearch should be invoked to display the current state of the search. Specifically, it should highlight the elements between the indexes low and high with a green background, indicate the element at the midpoint in magenta, and display the positions of the indexes low and mid (for instance below the line showing the array).

For a possible skeleton of the solution and additional details, you can refer to the material provided in Lab 12.

Exercise 7 [20 Points]: Comparing the number of comparissons required by linear and binary search.

Write a program that receives two positive integers as a command-line argument: the size and the step, such that size is smaller than 50,000 and step is smaller than size. The program should first initialize an array of the given step size with the first step natural numbers. Then, the program should perform both linear and binary search for every element, computing the following statistics:

- The minimum number of required comparisons
- The maximum number of comparisons
- The average number of comparisons

Additionally, it should compute the number of comparisons required for an element that is not present in the array (for instance, for number equal to size) Then, program should continue iterating by incrementing the array size exponentially.
